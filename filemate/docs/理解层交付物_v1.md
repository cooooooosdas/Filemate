## 三、Prompt 文件说明

### 3.1 classify.md — 分类 Prompt

**功能：** 让 AI 判断文件属于哪个类别
**输出格式：** JSON `{"category", "confidence", "course_name", "reason"}`
**分类规则：** 5 条判断规则（优先看内容、区分课件和作业、区分竞赛和考试等）
**当前版本：** v1，待迭代

### 3.2 extract.md — 实体抽取 Prompt

**功能：** 从文本中提取 5 个关键字段
**输出字段：**
- `course_name` — 课程名称
- `task_description` — 任务描述（≤30 字）
- `deadline` — 截止日期（YYYY-MM-DD）
- `location` — 地点
- `extra_entities` — 其他有用信息（字典）
**当前版本：** v1

### 3.3 milestone.md — 多里程碑识别 Prompt

**功能：** 从竞赛/大创通知中提取所有关键时间节点
**输出格式：** JSON 数组 `[{"event", "date", "order"}]`
**触发条件：** 仅竞赛通知和大创通知时调用
**当前版本：** v1

### 3.4 naming.md — 命名规范说明

**功能：** 命名生成器的规则说明（非 Prompt，命名器为规则实现）
**命名模板：** `[课程]-[类型]-[任务]-[截止]-[状态]`
**文件名最大长度：** 80 字符
**当前版本：** v1

### 6.5 错误分析

**已知问题：**

1. **关键词规则误判** — 23 次规则命中中有 9 次分错，规则置信度固定 83%，区分度不足
2. **.doc 解析失败** — 旧版 Word 格式（.doc）未被注册到解析器，2 个文件解析失败
3. **参考资料全部误判为作业** — "习题"、"练习"等关键词在参考资料和作业之间重叠
4. **待确认全部误判** — 行政通知被分到作业/考试通知，缺乏行政类关键词
5. **LLM 调用太少** — 28 份文件中只有 3 份走了 LLM，其余被关键词规则抢先拦截

### 6.6 v3 改进方向

1. 精减关键词，去掉类别间的重叠词
2. 降低规则置信度（83% → 70% 以下），让更多文件走 LLM
3. 修复 .doc 格式解析
4. 给待确认类别加行政类关键词（申请表、招募、统计工作等）
5. 样本总量已 28 份，接近 30 份目标；大创通知类别暂不补充样本
