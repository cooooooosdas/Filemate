# 命名生成规范 v1

## 命名模板
```
[课程名]-[文件类型]-[任务描述]-[截止时间]-[状态]
```

## 各字段说明

| 字段 | 说明 | 示例 |
|---|---|---|
| 课程名 | 文件所属的课程名称 | "操作系统"、"高等数学" |
| 文件类型 | 见分类体系 | "课件"、"作业"、"竞赛通知" |
| 任务描述 | 文件的核心内容简述，≤20 字 | "实验三"、"第三章习题" |
| 截止时间 | 有日期时转为 MMDD 格式，无截止日期填"待定" | "0415"（表示4月15日） |
| 状态 | 文件当前状态 | "待处理" |

## 完整示例

```
[操作系统]-[作业]-[实验三]-[待定]-[待处理]
[高等数学]-[参考资料]-[第三章]-[待定]-[待处理]
[数学建模]-[竞赛通知]-[报名截止]-[0415]-[待处理]
```

## 规则

1. 文件名总长度不超过 80 字符（不含扩展名）
2. 课程名过长时截断为前 10 个字
3. 任务描述过长时截断为前 10 个字
4. 截止日期统一为 MMDD 格式（如 0415 表示 4 月 15 日），无截止日期填"待定"
5. 不包含任何非法字符（\ / : * ? " < > |）
6. 如果某个字段无法提取，使用默认值：课程名="未分类"，任务="未命名"，截止="待定"

## 实现方式
命名生成器为规则实现，不调用 LLM。仅当任务描述超过 20 字时，调用 LLM 精简到 15 字以内。
